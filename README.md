# Event Manager (Calendar Pro) - Fullstack Application

This is a fullstack application for the Event Manager (Calendar Pro) project, built with React and Spring Boot. It provides a comprehensive user interface to manage personal events, scheduling, and calendar functionalities, ensuring you never miss a beat.

## 🔗 Live Demo
- **Frontend App**: [https://event-manager-pogwhu8n5-kaviyashreegs-projects.vercel.app](https://event-manager-pogwhu8n5-kaviyashreegs-projects.vercel.app)
- **Backend API**: [https://learning-backend-qwh4.onrender.com](https://learning-backend-qwh4.onrender.com)

## 🚀 Key Features

- **User Authentication**: Secure Login and Signup functionality.
- **Event Management**: Create, view, update, and delete (CRUD) operations for your events. Mark tasks as completed.
- **Interactive Calendar View**: A visual monthly calendar to easily oversee your upcoming events.
- **Event List View**: A detailed list format for managing tasks and events.
- **Notification System**: Real-time toast notifications for user actions (e.g., successful login, event updates).
- **Reminders**: A built-in Reminder Manager to alert you of approaching events.
- **Local Data Persistence**: Utilizes browser `localStorage` to securely save user data and event lists across sessions.

## 🛠️ Technology Stack

- **React 19**: Core library for building user interfaces.
- **Testing Library & Jest**: Frameworks for robust unit and component testing.
- **Vercel / GitHub Pages**: Configured for seamless deployment.

## 🏃‍♂️ Running Locally

In the project directory, you can run the following scripts:

### `npm install`
Installs all the required dependencies to run the frontend application.

### `npm start`
Runs the app in development mode.
Open [http://localhost:3000](http://localhost:3000) to view it in your browser. The page will automatically reload when you make changes.

### `npm test`
Launches the test runner in interactive watch mode for running unit tests.

### `npm run build`
Builds the app for production to the `build` folder. It correctly bundles React in production mode and optimizes the build for the best performance. The build is minified, and the filenames include hashes. Your app is ready to be deployed!
